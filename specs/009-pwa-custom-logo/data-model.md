# Data Model: PWA Manifest Structure

## Overview
For this feature, the data model refers to the structure of the `manifest.webmanifest` file generated by `vite-plugin-pwa`.

## Web App Manifest

The manifest will contain the following key-value pairs:

| Field | Value / Constraint | Description |
|-------|-------------------|-------------|
| `name` | "DailyGrain" | The full name of the application. |
| `short_name` | "DailyGrain" | The name displayed on the home screen. |
| `description` | "Master Your Time, Grain by Grain." | A description of what the app does. |
| `theme_color` | `#ffffff` | The color of the tool bar. |
| `background_color` | `#ffffff` | The background color for the splash screen. |
| `display` | `standalone` | Opens the app without browser UI. |
| `start_url` | `/` | The URL that loads when the app is launched. |
| `id` | `/` | Unique identifier for the app. |

## Icon Assets Schema

The `icons` array in the manifest must contain:

| Size | Type | Purpose | Destination Path |
|------|------|---------|------------------|
| 192x192 | `image/png` | `any` | `/icons/icon-192x192.webp` |
| 512x512 | `image/png` | `any` | `/icons/icon-512x512.webp` |
| 512x512 | `image/png` | `maskable` | `/icons/maskable-icon.webp` |

## Auxiliary Assets

These are not in the `icons` array but are referenced in `includeAssets` or HTML:

| Asset | Usage | Path |
|-------|-------|------|
| Apple Touch Icon | iOS Home Screen | `/icons/apple-touch-icon.webp` |
| Favicon | Browser Tab | `/favicon.ico` or `/vite.svg` (replaced) |
