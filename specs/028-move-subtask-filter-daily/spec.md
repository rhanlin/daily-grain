# Feature Specification: Daily Plan Gestures & Filters

**Feature Branch**: `028-move-subtask-filter-daily`  
**Created**: 2026-03-19  
**Status**: Draft  
**Input**: User description: "我要添加 2 個功能：在'每日計劃'頁面中：1. 已經存在每日計劃裡的 subTask 要可以透過手勢拖曳移動到隔天; 2. 新增一個 toggle 或是同值性操作方式，在UX 最佳化的前提之下，我要可以過濾掉每日任務顯示剩餘的項目;"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 手勢拖曳移動子任務至其他日期 (Priority: P1)

使用者在「每日計劃」頁面中，可以選中一個已排程的子任務，透過手勢將其拖曳至日期導覽的「前一天」或「後一天」按鈕，快速將該任務移動至目標日期。

**Why this priority**: 這是頻繁發生的操作，手勢化能大幅提升規劃的流暢度，符合 UX 最佳化目標。

**Independent Test**: 使用者長按並拖動一個子任務，將其移至上方日期列的「前一天」或「下一天」箭頭處並懸停，視圖應自動跳轉，放開後任務成功移至該日。

**Acceptance Scenarios**:

1. **Given** 使用者位於 2026-03-20 的計畫頁面，**When** 拖動子任務 A 並放置於「前一天」熱區，**Then** 子任務 A 移至 03-19；若放置於「下一天」熱區，**Then** 移至 03-21。
2. **Given** 正在拖動中，**When** 使用者取消拖動（在非熱區放開），**Then** 子任務應彈回原位，不發生日期變更。

---

### User Story 2 - 過濾每日任務 (Priority: P1)

使用者在「每日計劃」頁面中，可以透過一個切換開關（Toggle），在「顯示所有」與「僅顯示單次/多次任務」之間切換，暫時隱藏那些每天都會重複出現的例行性任務。

**Why this priority**: 當每日任務過多時，容易遮蔽今天真正重要的獨特任務。過濾功能能幫助使用者聚焦於「非例行性」的事項。

**Independent Test**: 開啟過濾開關後，所有型別為「每日 (Daily)」的任務從列表中消失，僅剩「單次 (One-time)」或「多次 (Multi-time)」任務。

**Acceptance Scenarios**:

1. **Given** 列表包含 2 個每日任務與 3 個單次任務，**When** 開啟「隱藏每日任務」開關，**Then** 列表僅顯示 3 個單次任務。
2. **Given** 過濾狀態已開啟，**When** 切換日期時，**Then** 過濾狀態應維持（或依據 UX 決定是否重置）。

---

### Edge Cases

- **跨月/跨年移動**: 拖曳至隔天時，若當前是月底，應正確計算下一天為下個月 1 號。
- **拖曳衝突**: 若目標日期已存在完全相同的子任務，系統應如何處理？（預設為允許重複）。
- **過濾後為空**: 若開啟過濾後清單變為空，應顯示適當的「所有項目已被隱藏」提示，而非空白一片。
- **拖曳中日期預覽**: 拖曳至箭頭熱區並懸停 500ms 時，系統應自動切換日期視圖至隔天，提供即時預覽回饋。

## Functional Requirements

- **FR-001**: 系統必須識別拖曳行為的目標區域。當拖曳開始時，畫面左右兩側應自動顯示半透明引導面板（標註「前一天/後一天」）作為熱區目標。
- **FR-002**: 系統必須支持將 `DailyPlanItem` 的日期屬性更新為目標日期（前一天或後一天）。
- **FR-003**: 系統必須在頂部 Toolbar 提供一個 UI 開關來控制過濾狀態。
- **FR-004**: 過濾邏輯必須基於子任務的 `type` 屬性（過濾掉 `daily` 類型）。
- **FR-005**: 系統必須確保在過濾狀態下，手動排序（Drag & Drop 排序）仍能正確運作，且不破壞隱藏項目的排序順序。

### Key Entities *(include if feature involves data)*

- **DailyPlanItem**: 包含 `date` 屬性，需被手勢操作更新。
- **SubTask**: 包含 `type` 屬性，作為過濾的判斷基準。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 調整任務日期的操作步驟從「點擊選單 -> 選擇日期 -> 確認」簡化為「一次拖曳完成」。
- **SC-002**: 列表過濾切換應在 100ms 內完成，無明顯卡頓。
- **SC-003**: 誤觸率降低，拖曳熱區的判定範圍需精確且有視覺回饋。
